# 디지털 가승(家乘) 플랫폼 기획서

---

## 1. 프로젝트 개요

### 1.1 배경 및 필요성

전통 족보(族譜)와 가승(家乘)은 한 가문의 혈통과 역사를 기록하는 소중한 문화유산이다. 그러나 현재의 종이 족보는 다음과 같은 한계를 가진다:

- **접근성 부족**: 한문과 세로쓰기 형식으로 되어 있어 젊은 세대가 읽기 어려움
- **갱신의 어려움**: 새로운 가족 구성원(출생, 혼인 등)이 생겨도 즉시 반영하기 어려움
- **단일 매체**: 텍스트 중심으로 사진, 영상, 음성 등 멀티미디어를 담을 수 없음
- **보존 위험**: 물리적 훼손, 분실의 위험이 상존함
- **공유 불편**: 가문 구성원 전체가 함께 열람하고 참여하기 어려움

### 1.2 프로젝트 목표

**"우리 가문의 이야기를, 누구나 쉽게 읽고 함께 채워가는 살아있는 가승"**

네이버 밴드, 카카오톡과 같은 친숙한 UX를 기반으로, 전통 족보/가승의 개념과 격식을 유지하면서도 사진, 영상, 음성을 자유롭게 담을 수 있는 디지털 가승 플랫폼을 구축한다.

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| **전통과 현대의 조화** | 족보의 세계(世系) 체계를 유지하되, 현대적 인터페이스로 제공 |
| **참여형 기록** | 가족 구성원 누구나 자신의 이야기를 직접 등록 |
| **멀티미디어 가승** | 글뿐 아니라 사진, 영상, 음성으로 가문의 생생한 기억을 보존 |
| **세대 연결** | 어르신의 구술 기록부터 아이의 성장 기록까지 세대를 잇는 플랫폼 |

## 2. 족보/가승 핵심 개념 정의

디지털 플랫폼에서도 반드시 반영해야 할 전통 족보/가승의 핵심 개념:

### 2.1 기본 용어

| 용어 | 한자 | 설명 | 플랫폼 반영 방식 |
|------|------|------|------------------|
| **시조(始祖)** | 始祖 | 가문의 첫 조상 | 가계도 최상단에 표시, 별도 소개 페이지 |
| **세(世)** | 世 | 시조로부터의 대수 | 각 인물 프로필에 "00세" 자동 표기 |
| **파(派)** | 派 | 가문 내 분파 | 파별 필터링 및 색상 구분 |
| **항렬(行列)** | 行列 | 같은 세대의 돌림자 | 세대 그룹 표시 시 항렬자 안내 |
| **배위(配位)** | 配位 | 배우자 | 인물 프로필에 배우자 정보 연결 |
| **묘소(墓所)** | 墓所 | 묘지 위치 | 지도 연동으로 위치 표시 |
| **본관(本貫)** | 本貫 | 성씨의 발상지 | 가문 대표 페이지에 표시 |

### 2.2 가승 기록 구조

전통 가승에 기록되는 정보를 디지털로 변환:

```
[전통 가승 항목]          →  [디지털 가승 항목]
─────────────────────────────────────────────
이름(諱), 자(字), 호(號)  →  프로필: 이름, 호, 별명
생년월일(음/양력)          →  생년월일 (음양력 자동 변환)
졸년월일                   →  기일 (기일 알림 기능)
배위 및 외가 정보          →  배우자 프로필 연결, 외가 링크
자녀 기록                  →  가계도 자동 연결
묘소 위치(좌향 포함)       →  지도 + 사진 + 방문 기록
관직/학력/경력             →  이력 타임라인
행장(行狀)/유훈(遺訓)      →  멀티미디어 스토리 (영상, 음성, 글)
```

## 3. 주요 기능

### 3.1 가계도 (세계도, 世系圖)

**전통 족보의 핵심을 시각적으로 구현**

- **인터랙티브 가계도**: 터치/클릭으로 확대, 축소, 이동 가능한 트리 뷰
- **세대(世) 표시**: 각 인물 옆에 몇 세인지 자동 계산 표시
- **파(派) 구분**: 파별 색상 구분, 특정 파만 필터링하여 보기
- **인물 카드**: 가계도에서 인물을 탭하면 사진, 기본 정보가 팝업으로 표시
- **검색**: 이름, 항렬, 세대로 특정 인물 빠르게 찾기
- **가로/세로 보기**: 전통 족보식(세로) / 현대식(가로) 전환 가능

### 3.2 인물 프로필

**한 사람의 일생을 한 페이지에**

각 가족 구성원의 상세 페이지:

- **기본 정보**: 이름(한글/한자), 자, 호, 세대(世), 생년월일(음/양력), 본관
- **프로필 사진**: 대표 사진 및 시대별 사진 갤러리
- **가족 관계**: 부모, 배우자, 자녀를 탭하면 해당 인물로 이동
- **생애 타임라인**: 출생 → 학력 → 경력 → 주요 사건을 시간순 정리
- **멀티미디어 기록**:
  - 사진첩: 생애 사진, 가족 사진
  - 영상: 인터뷰, 행사 영상, 일상 기록
  - 음성: 육성 녹음, 구술 기록, 좋아하던 노래
  - 글: 유훈, 편지, 일기, 회고록
- **묘소 정보**: 위치(지도), 좌향, 사진, 성묘 기록

### 3.3 가문 스토리 (타임라인)

**네이버 밴드처럼 가문의 소식을 공유하는 공간**

- **소식 피드**: 가족 구성원이 사진, 영상, 글을 자유롭게 올리는 타임라인
- **경조사 알림**: 생일, 기일, 결혼기념일 등 자동 알림
- **행사 기록**: 제사, 시제, 종친회 모임 등의 사진/영상 아카이브
- **댓글 및 반응**: 가족 간 소통 (좋아요, 댓글)
- **공지사항**: 종중 공지, 문중 행사 안내

### 3.4 구술 기록 (어르신 인터뷰)

**사라져가는 기억을 살아있는 기록으로**

- **음성/영상 녹음**: 앱 내에서 바로 녹음/촬영
- **자동 텍스트 변환**: 음성을 텍스트로 자동 변환 (STT)
- **질문 가이드**: "어린 시절 이야기", "결혼 이야기" 등 인터뷰 질문 템플릿 제공
- **태그 연결**: 녹음 내용에 등장하는 인물을 가계도와 연결

### 3.5 기일/제사 관리

**전통 제례 문화의 디지털 지원**

- **기일 캘린더**: 음력/양력 자동 변환, 매년 알림
- **제사 참석 관리**: 참석 여부 확인 (카톡 투표처럼)
- **제사 정보**: 장소, 시간, 준비물, 지방(紙榜) 작성 가이드
- **성묘 기록**: 방문 사진, 묘소 관리 현황

### 3.6 문서 보관소

**종이 족보의 디지털 아카이브**

- **기존 족보 스캔**: 종이 족보를 촬영/스캔하여 보관
- **고문서 보관**: 가문 관련 고문서, 교지, 편지 등
- **OCR 지원**: 스캔한 문서에서 텍스트 추출 (한자 포함)

## 4. 사용자 역할 및 권한

### 4.1 역할 체계

| 역할 | 설명 | 주요 권한 |
|------|------|-----------|
| **대종회 관리자** | 전체 족보 관리 총괄 | 모든 권한, 가계도 구조 수정, 회원 승인 |
| **파 관리자** | 각 파(派)의 관리자 | 해당 파 내 인물 정보 수정, 회원 승인 |
| **가족 구성원** | 일반 가족 회원 | 자신/직계 프로필 수정, 스토리 작성 |
| **열람자** | 읽기 전용 | 가계도 열람, 스토리 읽기 |

### 4.2 가입 및 인증

- 기존 가족 구성원의 **초대 링크**를 통해 가입
- 관리자가 가계도상의 위치를 **확인 후 승인**
- 카카오/네이버 소셜 로그인 지원

## 6. 기술 구성

### 6.1 플랫폼

| 구분 | 기술 | 비고 |
|------|------|------|
| **프론트엔드** | React Native (모바일) / React (웹) | iOS, Android, 웹 동시 지원 |
| **백엔드** | Node.js 또는 Python (Django/FastAPI) | REST API + WebSocket |
| **데이터베이스** | PostgreSQL + Neo4j | 관계형 데이터 + 가계도 그래프 |
| **파일 저장소** | AWS S3 / Google Cloud Storage | 사진, 영상, 음성 파일 |
| **음성 변환** | Whisper API / Clova Speech | 구술 기록 STT |
| **지도** | 카카오맵 / 네이버 지도 API | 묘소 위치 표시 |
| **알림** | FCM (Firebase Cloud Messaging) | 기일 알림, 소식 알림 |
| **음양력 변환** | 한국천문연구원 API | 음력/양력 자동 변환 |

### 6.2 시스템 아키텍처 (개요)

```
[모바일 앱 / 웹]
       │
       ▼
  [API Gateway]
       │
  ┌────┴────┐
  ▼         ▼
[API 서버]  [미디어 서버]
  │              │
  ▼              ▼
[PostgreSQL]  [S3 스토리지]
[Neo4j]       (사진/영상/음성)
```

## 7. 데이터 모델 (핵심)

### 7.1 인물 (Person)

```
Person {
  id                  : UUID
  name_korean         : String       // 이름 (한글)
  name_hanja          : String       // 이름 (한자)
  courtesy_name       : String       // 자(字)
  pen_name            : String       // 호(號)
  generation          : Integer      // 세(世) - 시조로부터 대수
  generation_char     : String       // 항렬자
  birth_date_solar    : Date         // 양력 생년월일
  birth_date_lunar    : Date         // 음력 생년월일
  death_date_solar    : Date         // 양력 졸년월일
  death_date_lunar    : Date         // 음력 졸년월일
  gender              : Enum         // 성별
  branch              : String       // 파(派)
  profile_photo       : URL          // 대표 사진
  bio                 : Text         // 약력
  burial_location     : GeoPoint     // 묘소 위치
  burial_direction    : String       // 좌향
  is_living           : Boolean      // 생존 여부
}
```

### 7.2 관계 (Relationship)

```
Relationship {
  id          : UUID
  person_id   : UUID       // 기준 인물
  related_id  : UUID       // 관련 인물
  type        : Enum       // 부자, 모자, 부부, 형제 등
}
```

### 7.3 미디어 (Media)

```
Media {
  id          : UUID
  person_id   : UUID       // 연결된 인물
  type        : Enum       // photo, video, audio, document
  url         : URL        // 파일 경로
  title       : String     // 제목
  description : Text       // 설명
  date        : Date       // 촬영/녹음 일자
  uploaded_by : UUID       // 업로드한 사람
  created_at  : DateTime
}
```

### 7.4 스토리 (Story)

```
Story {
  id          : UUID
  author_id   : UUID       // 작성자
  content     : Text       // 본문
  media       : [Media]    // 첨부 미디어
  tagged      : [Person]   // 태그된 인물
  type        : Enum       // 일상, 경조사, 제사, 공지
  created_at  : DateTime
  comments    : [Comment]
  reactions   : [Reaction]
}
```

## 8. 개발 로드맵

### Phase 1: MVP (3개월)

- 가계도 기본 구현 (트리 뷰, 인물 등록)
- 인물 프로필 (기본 정보 + 사진)
- 사용자 가입/인증 (초대 기반)
- 웹 버전 우선 개발

### Phase 2: 멀티미디어 (2개월)

- 영상/음성 업로드 및 재생
- 구술 기록 기능 (녹음 + STT)
- 사진 갤러리 강화

### Phase 3: 소셜 기능 (2개월)

- 스토리 타임라인 (피드)
- 댓글, 반응
- 기일 캘린더 및 알림
- 경조사 알림

### Phase 4: 고도화 (2개월)

- 모바일 앱 (iOS, Android)
- 기존 종이 족보 스캔/OCR
- 묘소 지도 연동
- 파별 관리 기능

## 9. 차별화 포인트

### 기존 족보 서비스 vs 본 플랫폼

| 구분 | 기존 족보 서비스 | 본 플랫폼 |
|------|-----------------|-----------|
| 형태 | 텍스트 중심 DB | 멀티미디어 가승 |
| UX | 사무적, 딱딱함 | 밴드/카톡처럼 친근함 |
| 참여 | 관리자만 수정 | 가족 누구나 참여 |
| 소통 | 없음 | 스토리, 댓글, 반응 |
| 미디어 | 사진 일부 | 사진, 영상, 음성 |
| 기억 보존 | 텍스트 기록만 | 어르신 구술 녹음, 육성 보존 |
| 세대 연결 | 어른 세대 중심 | 전 세대가 함께 사용 |

## 10. 기대 효과

1. **세대 간 소통 강화**: 젊은 세대가 자연스럽게 가문의 역사에 관심을 갖게 됨
2. **살아있는 기록**: 텍스트를 넘어 어르신의 육성, 표정, 이야기가 영원히 보존됨
3. **가문 결속력 강화**: SNS형 소통으로 멀리 사는 친척과도 자연스럽게 교류
4. **전통 계승**: 기일, 제사, 성묘 등 전통 문화를 디지털로 이어감
5. **역사 보존**: 종이 족보의 디지털 백업으로 영구 보존 가능
